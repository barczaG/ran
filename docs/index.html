linalg.Vector([arg])
Class representing a real vector.

Parameters

nametypedescription
argnumber Array VectorThe constructor argument. If it is a number, it sets the dimension of the vector. If it is an array, the vector is initialized with the array elements. If it is another vector, it is copied to this vector. If not specified, a 3D vector is created directing in the X axis.
linalg.Vector.v()
Returns the vector as an array.

Returns

Array
The vector as an array.

Example

let vec = new ran.linalg.Vector(3)
vec.v()
// => [1, 0, 0]

mc.MCMC(logDensity[, config[, initialState]])
Class implementing a Markov chain Monte Carlo sampler. MCMC samplers can be used to approximate integrals by efficiently sampling a density that cannot be normalized or sampled directly.

Parameters

nametypedescription
logDensityFunctionThe logarithm of the density function to estimate.
configObjectObject describing some configurations. Supported properties:
  • dim: Dimension of the state space to sample. Default is 1.
  • maxHistory: Maximum length of history for aggregated computations. Default is 1000.
initialStateObjectThe initial internal state of the sampler. Supported properties: x (the starting state), samplingRate (sampling rate) and internal for the child class' own internal parameters.
mc.MCMC.state()
Returns the current state of the sampler. The return value of this method can be passed to a sampler of the same type to continue a previously warmed up sampler.

Returns

Object
Object containing all relevant parameters of the sampler.
mc.MCMC.statistics()
Computes basic statistics of the sampled state variables based on historical data. Returns mean, standard deviation and coefficient of variation.

Returns

Object[]
Array containing objects for each dimension. Objects contain mean, std and cv.

Computes acceptance rate based on historical data.

mc.MCMC.ac()
Computes the auto-correlation function for each dimension based on historical data.

mc.MCMC.warmUp(callback[, maxBatches])
Carries our the initial warm-up phase of the sampler. During this phase, internal parameters may change and therefore sampling does not take place. Instead, all relevant variables are adjusted.

Parameters

nametypedescription
callbackFunctionCallback function to call when an integer percentage of the warm-up is done. The percentage of the finished batches is passed as a parameter.
maxBatchesnumberMaximum number of batches for warm-up. Each batch consists of 10K iterations. Default values i 100.

Performs the sampling of the target density. Note that during sampling, no parameter adjustment is taking place.
